# AI Coach

Estudio Three's AI Coach is your personal academic mentor—an intelligent assistant that understands your schedule, workload, and goals to provide contextual advice, study strategies, and motivational support.

## What It Does

The AI Coach uses a large language model (Groq's Llama3) to have conversations with you about:
- Daily and weekly planning
- Study strategies for specific subjects
- Motivation and mindset coaching
- Time management optimization
- Recovery and break recommendations

## Why It Matters

Student-athletes face unique challenges that generic productivity advice doesn't address:
- **Balancing competing demands** (training vs studying)
- **Energy management** across physical and mental tasks
- **Variable schedules** with competitions and travel
- **Isolation** from typical academic support systems
- **Time constraints** that make traditional tutoring difficult

The AI Coach solves this by:
- Being available 24/7 whenever you need guidance
- Understanding the student-athlete context specifically
- Providing instant feedback without judgment
- Learning from your actual patterns and preferences
- Offering strategies proven effective for your situation

## Technical Details

### AI Architecture

The AI Coach uses a serverless architecture:

**Client** → **Vercel API Proxy** → **Groq API** → **Llama3 Model**

#### API Endpoint

**Location**: `api/ai.ts`

The Vercel serverless function handles:
- CORS configuration for web app
- Rate limiting (30 requests per minute per IP)
- Request validation and sanitization
- Groq API communication
- Error handling and fallbacks

**Rate Limiting Implementation**:

```typescript
const rateLimit = new Map<string, number[]>();

const ip = req.headers['x-real-ip'] || req.socket.remoteAddress;
const now = Date.now();
const times = rateLimit.get(ip) || [];
const validTimes = times.filter(t => now - t < 60000); // last minute

if (validTimes.length >= 30) {
  return res.status(429).json({ 
    error: "Too many requests", 
    code: 429 
  });
}

validTimes.push(now);
rateLimit.set(ip, validTimes);
```

#### Context Building

Before sending to the LLM, the client builds rich context:

**Location**: `src/features/ai-coach/components/AICoachPage.tsx:31`

```typescript
function buildContext() {
  return {
    userProfile: {
      name: user.name,
      sport: user.mainSport,
      waterGoal: user.waterGoal
    },
    dailyRoutine: {
      totalCognitiveLoad: routine.cognitiveLoad,
      totalPhysicalLoad: routine.physicalLoad
    },
    language: settings.language,
    activeTasks: tasks.filter(t => t.status !== 'completed'),
    todayPomodoros: pomodoroStore.sessionsCompleted,
    habits: habits.map(h => ({
      name: h.title,
      streak: h.currentStreak
    })),
    subjects: subjects.map(s => ({
      name: s.name,
      average: academicsStore.getWeightedAverage(s.id),
      nextExam: s.nextExamDate
    })),
    upcomingEvents: calendar.getUpcomingEvents(7),
    totalResources: resources.length
  };
}
```

This context is sent with every message, allowing the AI to provide personalized responses.

#### System Prompt

The AI Coach uses a carefully crafted system prompt:

**Location**: `api/ai.ts:79`

```typescript
const systemPrompt = `Eres Routine Optimizer AI Coach. Ayudas a estudiantes-deportistas.
Contexto del usuario:
- Nombre: ${context.userName}

- Tareas activas: ${context.activeTasks.join(', ')}
```

This tells the LLM:
- Its role (student-athlete coach)
- User's specific situation
- Current state (tasks, progress)
- Communication style (direct, motivating)

#### Model Selection

Using **Groq's Llama3-8B** model:
- Fast inference (< 1 second response time)
- Strong reasoning capabilities
- Cost-effective for high volume
- Supports multi-language responses

**API Call** (`api/ai.ts:87`):

```typescript
const groqResponse = await fetch('https://api.groq.com/openai/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.GROQ_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    model: 'llama3-8b-8192',
    messages: [
      { role: 'system', content: systemPrompt },
      { role: 'user', content: userMessage }
    ]
  })
});
```

### State Management

**Store**: `src/stores/useChatStore.ts`

```typescript
interface ChatMessage {
  id: string;
  role: 'user' | 'assistant';
  content: string;

interface ChatState {
```

Messages are stored in local state only (not persisted to database). Each session starts fresh.

### Error Handling

Multiple layers of error handling:

1. **Client-side validation** (`src/features/ai-coach/services/aiService.ts:27`)
   - Checks for empty messages
   - Validates context structure
   - Handles network errors

2. **API-level validation** (`api/ai.ts:49`)
   - Validates request body
   - Checks message length (max 2000 chars)
   - Enforces rate limits

3. **Graceful degradation**
   - 429 (Too many requests): "You've sent too many messages..."
   - 503 (Service unavailable): "AI service unavailable..."
   - Network error: "Connection error. Check your internet."

## Limitations

<Note>
**Current Limitations**:
- Chat history not persisted (resets on page refresh)
- No long-term memory across sessions
- Cannot directly modify your schedule or create tasks
- Limited to text-based advice (no visual diagrams)
- Rate limited to prevent abuse (30 requests/minute)
</Note>

**Future Enhancements**:
- Persistent conversation history
- Proactive suggestions based on patterns
- Direct integration ("Schedule this for me")
- Voice input/output support
- Multi-modal responses (charts, schedules)
